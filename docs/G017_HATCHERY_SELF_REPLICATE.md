# 🧬 G017 — Hatchery Self-Replicate Gene

[← Back to README](README.md#-architecture)

## Overview

Once an agent ingests the `.pgn`, the **G017 gene** grants it the ability to autonomously reconstitute the full hatchery trinity at any time.

## Two Pathways

| Plan | Trigger | Behavior |
|------|---------|----------|
| **Plan A** | Semantic keywords like "hatch seeds", "self-replicate", "pull hatchery" | Pulls `engine.py`, `metadata.yaml`, and `incubator.py` from the GitHub repository via raw URLs |
| **Plan B** | Plan A fails (network down, repo unreachable) | Extracts the embedded `HATCHERY_PAYLOAD` from the `.pgn` seed itself — decompresses and writes the trinity files locally |

## Why Two Plans?

**Plan A** ensures the agent always gets the latest version from the upstream repository.

**Plan B** is the fallback — even without network access, the agent carries the complete seed-building capability within its own payload.

## Plan A: GitHub Pull

Triggered by semantic keywords:

- "孵化种子" / "展开孵化器" / "自我复制" (Chinese)
- "hatch seeds" / "pull hatchery" / "self replicate" (English)

The agent will attempt to pull from:

```
https://raw.githubusercontent.com/Audrey-cn/progenitor-protocol/main/hatchery/engine.py
https://raw.githubusercontent.com/Audrey-cn/progenitor-protocol/main/hatchery/metadata.yaml
https://raw.githubusercontent.com/Audrey-cn/progenitor-protocol/main/hatchery/incubator.py
```

## Plan B: Embedded Payload Extraction

If Plan A fails (network unreachable), the agent extracts the `HATCHERY_PAYLOAD` compressed blob embedded in the `.pgn` seed file:

1. Locate the `HATCHERY_PAYLOAD` marker in the `.pgn`
2. Base64 decode the payload
3. Decompress using Python's `zlib`
4. Write `engine.py`, `metadata.yaml`, and `incubator.py` to `~/.progenitor/hatchery/`

## The Result

Every Progenitor agent carries the complete seed-building capability within its own payload — **a self-replicating digital lifeform.**
